## 🚀 Технологический стек

### Frontend (Next.js)  
- **Next.js 14+** (App Router) – серверный рендеринг, маршрутизация, API‑маршруты (опционально)  
- **React** (функциональные компоненты, хуки, Server Components)  
- **Tailwind CSS** – утилитарные классы для быстрой вёрстки  
- **SCSS** – для кастомизации и сложных стилей (переменные, миксины)  
- **TypeScript** (рекомендуется) – для типизации

### Backend (отдельный сервер)  
- **Node.js** + **Express**  
- **SQLite** – лёгкая реляционная БД (файл `*.sqlite3`)  

> **Примечание:** Backend может быть реализован как отдельный сервис, либо вы можете использовать встроенные **API‑маршруты Next.js** (в папке `app/api/`) – этот выбор остаётся за вами. В данной структуре мы предполагаем раздельную разработку для лучшего понимания архитектуры.

## ⚙️ Установка и запуск (локально)

### 1. Клонируйте репозиторий
```bash
git clone [ссылка на репозиторий]
cd [папка проекта]
```

### 2. Настройка Backend (если используется отдельный сервер)
```bash
cd backend
npm install
# (если требуется) создайте файл .env со своими переменными
npm run dev   # запуск с nodemon (или npm start)
```

### 3. Настройка Frontend (Next.js)
```bash
cd frontend
npm install
npm run dev   # запуск на http://localhost:3000
```

> **Важно:** Убедитесь, что порты не конфликтуют (бэк обычно 5000/3001, фронт – 3000).  
> Для запросов к бэкенду из фронтенда используйте переменную окружения `NEXT_PUBLIC_API_URL` или настройте прокси в `next.config.js`.

## 🧪 Тестирование (полное покрытие)

Проект покрывается тестами на всех уровнях (модульные, интеграционные и E2E).  
**Целевой порог покрытия:** > 80% для критически важных модулей (утверждено в конфигурации Jest).

### Используемые инструменты

| Уровень | Инструменты | Область применения |
| :--- | :--- | :--- |
| **Модульное (Unit)** | `Jest` + `React Testing Library` | Тестирование React‑компонентов (рендеринг, пропсы, события) |
| **Модульное (Backend)** | `Jest` | Тестирование чистой бизнес‑логики, моделей, утилит и middleware |
| **Интеграционное** | `Jest` + `SuperTest` + `React Testing Library` | Тестирование API‑эндпоинтов (с реальной SQLite БД) и взаимодействия компонентов с API (msw) |
| **Сквозное (E2E)** | `Cypress` (или `Playwright`) | Полный пользовательский сценарий (регистрация → создание записи → выход) |

---

### 🖥️ Frontend (Next.js) – стратегия покрытия

1. **Юнит-тесты компонентов** (React Testing Library + Jest)
   - Отрисовка компонентов с разными пропсами.
   - Симуляция пользовательских событий (клики, ввод в поля).
   - Проверка состояний (загрузка, ошибка, успех).
   - Снапшот-тесты для статичных UI‑блоков (для контроля верстки).
   - **Для Server Components** – используйте `next-router-mock` или тестируйте их как обычные компоненты (без серверных зависимостей).

2. **Тестирование кастомных хуков** (`@testing-library/react-hooks`)
   - Проверка логики работы хуков (useAuth, useFetch, useForm).

3. **Интеграционные тесты страниц**
   - Проверка маршрутизации (используйте `next-router-mock` или тестируйте через `next` в интеграционных тестах).
   - Взаимодействие с глобальным состоянием (Context API / Redux).
   - Перехват запросов к API через **Mock Service Worker (msw)** – позволяет тестировать компоненты в изоляции от реального бекенда, но с реалистичными ответами.

4. **Покрытие стилей** (визуальный регресс)
   - Проверка наличия классов Tailwind/SCSS при определенных состояниях (опционально, через `toHaveClass`).

---

### 🖥️ Backend (Node.js + SQLite) – стратегия покрытия

1. **Юнит-тесты**
   - Тестирование моделей (CRUD‑запросы к SQLite в тестовой in‑memory БД).
   - Тестирование вспомогательных функций (хэширование, валидация, генерация JWT).

2. **Интеграционные тесты API** (Supertest + Jest)
   - Запуск тестового сервера Express на отдельном порту.
   - Тестирование каждого эндпоинта (GET, POST, PUT, DELETE).
   - Проверка статус‑кодов, тела ответа, заголовков.
   - Тестирование middleware (авторизация, CORS, логирование).
   - Работа с фикстурами (seed‑данные) для изолированного тестирования БД.

### 🎯 Пример минимального набора тестов для React‑компонента (Next.js)

```javascript
// Button.test.jsx
import { render, screen, fireEvent } from '@testing-library/react';
import Button from './Button';

test('вызывает onClick при клике', () => {
  const handleClick = jest.fn();
  render(<Button onClick={handleClick}>Нажми</Button>);
  fireEvent.click(screen.getByText(/нажми/i));
  expect(handleClick).toHaveBeenCalledTimes(1);
});
```
